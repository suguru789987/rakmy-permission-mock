# ラクミー 経営管理｜権限設定 モック

会社画面の権限設定UI（権限一覧／権限編集／指標カタログ／役割サマリ／ユーザー割当）の動作モック。

公開デモ: https://suguru789987.github.io/rakmy-permission-mock/

## デモモード（ヘッダー右上トグル）

2つのユースケースを切り替えて確認できる（選択は端末に保持）:

- **🆕 新規導入デモ** … 導入直後の状態。権限テンプレートで **推奨5権限（全権オーナー／本部管理／エリア長／店長／従業員）が登録済み**でランディング。明示バナーで「登録済み・確認→編集」を案内。会社ロールは **オーナー＋本部管理（母体）** の2つ（経理／人事は未作成）。
- **🏢 運用デモ** … 回っている状態。会社ロールは **オーナー＋経理／人事（本部管理を分割）**、カスタムロール追加も可。

差は「会社ロールが**本部管理（母体）のまま**か、**経理／人事に分割済み**か」。「母体を作って後で分ける／導入→運用」の流れを体現（経理・人事は運用で「＋新規作成」により母体から派生）。

### 権限テンプレートの推奨権限（自動登録）
| ロール | 区分 | 推奨権限 |
|---|---|---|
| オーナー（全権） | 会社 | 全権固定 |
| 本部管理 | 会社 | 全権相当（全社の設定・一覧を操作／閲覧） |
| エリア長 | 店舗（掌握店舗群） | 掌握店舗の設定・一覧を全て操作／閲覧 |
| 店長 | 店舗（自店舗） | 自店舗の設定・一覧を全て操作／閲覧 |
| 従業員 | 店舗（自店舗） | 閲覧中心（操作は最小） |

## 権限の作成・編集（統合画面）

新規作成と既存編集は**同じ画面**を兼用し、タイトルを出し分け:
- **＋ 権限を新規作成**（ヘッダー）→ 「権限を新規作成」（①ベース選択あり）
- 行の **「権限編集」**・内訳セル → 「権限編集：〇〇」（①ベース選択は隠し、現在の権限を②で直接編集→上書き保存）

### 3ステップ
1. **① ベースを選ぶ**（ハードプルダウン）。候補は**デモモードで出し分け**:
   - 新規導入デモ … ベーステンプレート（全権オーナー相当／本部管理／エリア長／店長／従業員）＋プレーンテンプレート（既存コピーは出さない）
   - 運用デモ … 上記＋**既存権限をコピー**（複製はカタログも引き継ぐ）
2. **② 権限内容を編集**。画面ごとに `操作／閲覧／なし` を直接編集（テンプレを土台に追加・削減）。**画面名クリックで列（機能・項目）ごとの内訳**を展開し、**列単位でも可否を編集**（種別ごとに なし/閲覧/操作/全て、常時は閲覧固定）。**画面の大元スイッチは配下の全列に連動**。上部に概要（許可数）。
3. **③ 基本設定**：権限名（必須）・区分（店舗/会社）・データ範囲（自店舗/指定店舗群/全社）。

- 区分・データ範囲はベースの既定。ベースと変えると**整合警告**を表示。
- 細かい機能・操作・指標は、一覧のカテゴリセルから**詳細編集**（カテゴリ詳細／指標カタログ）へ。
- 深い編集の入口（カテゴリ詳細・指標カタログ）は別画面として併存。

### ロールに画面（機能）権限を追加する
- **既存画面を付与**：一覧で対象画面の列を表示 → セルクリックで詳細編集、または行の「権限編集」で②グリッドから `なし→閲覧/操作` に変更。
- **画面内の機能・項目（列）単位**：権限編集②で画面名をクリックして内訳を展開し、各機能のセグメントで可否を設定。
- **分析画面の指標／タグ**：行の「指標カタログ／カタログ作成」で母集合を選定。
- ※ システムに無い**新しい画面カテゴリ自体の追加**はプロダクト側のマスタ機能（本モックの権限設定UIの対象外）。新画面は各ロールに既定「なし」で並び、上記で付与する。

## 画面構成

権限一覧の上部ナビは用途で2セットに分離:
- **権限の設定** … `権限編集` ／ `指標カタログ`
- **割当・俯瞰** … `役割サマリ` ／ `ユーザー割当`

## 権限一覧マトリクス

ロール × 権限カテゴリの一覧。**一覧画面権限 / 設定画面権限** の2タブで出し分け（各タブはその群のカテゴリのみ表示）。

- **一覧画面権限**（青）＝ 一覧・分析画面の閲覧可否
- **設定画面権限**（緑）＝ 設定画面の操作可否

左端は **区分列**（会社／店舗）を行にまたがり表示。権限名・操作列は固定幅でロックし、カテゴリ列が増えると横スクロール。

**オーナー（全権）は一覧から除外**：全権固定で編集対象外のため、マトリクスは設定対象の会社/店舗ロールのみ。オーナー専用機能（**カタログ管理／付与先を指定**）はヘッダーの「オーナー操作」群に集約。

### セル表示
- **一覧画面**：`閲覧`（見られる）／`一部`（一部のみ）／`—`（なし）
- **設定画面**：`操作`＋`閲覧可`（操作は閲覧を内包）／`閲覧可`（見るだけ）／`—`（なし）
- **「一部」** ＝ カテゴリ内の複数機能で設定が混在している状態。
- **セルをクリック** → そのロール×カテゴリの設定詳細へ直接遷移（オーナーは固定のため対象外）。
- 折りたたみ時の内訳セルは列名に沿った許可数（一覧＝`閲覧 N/総数`、設定＝`操作 N/総数`）。

### 表示ロール選択（タブの上）
- タブ（一覧/設定）の**上**に「👤 表示ロール」ハードプルダウン。**両タブ共通で効く行スコープ**。
- 大別**2グループ（会社ロール／店舗ロール）**＋内訳ロールを複数選択（オーナーは一覧対象外のため非表示）。
- グループ見出しで一括トグル、全て／クリア。階層は **行（ロール）→ 領域（タブ）→ 列（列表示）**。

### 列の表示（ツールバー）
- **列プルダウン**（表外固定・横スクロールで動かない）：`全て`＋各カテゴリを複数選択。クリア／閉じる、未選択は内訳1列。
- 既定で**主要画面の列を表示**（重要度順）。列順は選択順に従う。
- 列選択は `localStorage` に**自動記憶** → 次回アクセス時も同じ列（名前付き保存ビューは廃止）。

## 権限編集

- **一覧画面権限 / 設定画面権限** の2タブ。各タブ最上部に **全カテゴリ一括設定（初期値）** バー（一覧＝閲覧/なし、設定＝全て/操作/閲覧/なし）。
- カテゴリ単位で一括、「詳細 ＞」で機能・操作・指標を個別設定。混在は「一部」表示。
- 新規実装は ★★ バッジ。

### 権限レベル（機能の性質ごと）

権限は `なし ⊂ 閲覧 ⊂ 操作 ⊂ 全て` の順序（操作は閲覧を内包）。機能の種別で取りうる段階が異なる:

| 種別 | 段階 | 例 |
|---|---|---|
| 常時(base) | 閲覧固定 | （常時表示の閲覧） |
| 閲覧専用(view) | なし / 閲覧 | 分析・レポート・各種閲覧 |
| 設定(edit) | なし / 閲覧 / 操作 / 全て(削除含む) | 納品書・棚卸・従業員管理 など |
| 管理(admin) | なし / 閲覧 / 設定可 | 会社情報・お支払い・連携設定・ユーザー など |
| 実行(action) | なし / 実行可 | ダウンロード・DnD編集(店舗)・勤怠取込 |

- **選択UIの点灯**：`操作`（または`設定可`）を選ぶと下位の`閲覧`も同時点灯（操作⊃閲覧の可視化）。`全て`選択時は`全て`のみ点灯。
- **実行(action)** は「実行できる／できない」のアクションのため閲覧のみは設けない（2択）。

## ロール構成

- **オーナー** … 全権固定（編集不可）。全権の付与先を指名可。
- **会社ロール（全社・本部＝職務分掌）** … 経理担当・人事担当など。設定権限を職務に応じて付与。
- **店舗ロール（店舗現場）** … 店長・従業員・エリア長。自店舗スコープ、閲覧中心。

スコープ規則：店舗スコープの「従業員」は自店舗のみ／全社集計指標など会社スコープ専用は店舗ロールに付与不可。

## ユーザー割当

- ユーザーに権限ロールを割当（複数可）。複数ロールは権限が合算される。
- **有効範囲** 列＝スコープ（自店舗／全社／指定店舗群）を表示。未割当は「なし」。

## 権限の階層

1機能の中では `なし ⊂ 閲覧 ⊂ 操作 ⊂ 全て` の順序（操作は閲覧を内包）。表示専用画面と操作画面は別カテゴリとして分離（例：`従業員（閲覧）` と `従業員管理`）。
