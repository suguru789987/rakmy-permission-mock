## デモモード（ヘッダー右上トグル）

2つのユースケースを切り替えて確認できる（選択は端末に保持）:

- **🆕 新規導入デモ** … 導入直後の状態。権限テンプレートで **推奨5権限（全権オーナー／本部管理／エリア長／店長／従業員）が登録済み**でランディング。明示バナーで「登録済み・確認→編集」を案内。会社ロールは **オーナー＋本部管理（母体）** の2つ（経理／人事は未作成）。
- **🏢 運用デモ** … 回っている状態。会社ロールは **オーナー＋経理／人事（本部管理を分割）**、カスタムロール追加も可。

差は「会社ロールが**本部管理（母体）のまま**か、**経理／人事に分割済み**か」。「母体を作って後で分ける／導入→運用」の流れを体現（経理・人事は運用で「＋新規作成」により母体から派生）。

### 権限テンプレートの推奨権限（自動登録）
| ロール | 区分 | 推奨権限 |
|---|---|---|
| オーナー（全権） | 会社 | 全権固定 |
| 本部管理 | 会社 | 全権相当（全社の設定・一覧を操作／閲覧） |
| エリア長 | 店舗（掌握店舗群） | 掌握店舗の設定・一覧を全て操作／閲覧 |
| 店長 | 店舗（自店舗） | 自店舗の設定・一覧を全て操作／閲覧 |
| 従業員 | 店舗（自店舗） | 閲覧中心（操作は最小） |

## 権限の作成・編集（統合画面）

新規作成と既存編集は**同じ画面**を兼用し、タイトルを出し分け:
- **＋ 権限を新規作成**（ヘッダー）→ 「権限を新規作成」（①ベース選択あり）
- 行の **「権限編集」**・内訳セル → 「権限編集：〇〇」（①ベース選択は隠し、現在の権限を②で直接編集→上書き保存）

### 3ステップ
1. **① ベースを選ぶ**（ハードプルダウン）。候補は**デモモードで出し分け**:
   - 新規導入デモ … ベーステンプレート（全権オーナー相当／本部管理／エリア長／店長／従業員）＋プレーンテンプレート（既存コピーは出さない）
   - 運用デモ … 上記＋**既存権限をコピー**（複製はカタログも引き継ぐ）
2. **② 権限内容を編集**。画面ごとに `操作／閲覧／なし` を直接編集（テンプレを土台に追加・削減）。**画面名クリックで同一画面に「内訳（機能）→ 指標」をインライン展開**し編集（機能＝なし/閲覧/操作（登録・編集・削除を含む）、指標（閲覧詳細）＝各項目〔表示〕〔非表示〕、常時は閲覧固定）。**画面の大元スイッチは配下の全列に連動**。上部に概要（許可数）。
3. **③ 基本設定**：権限名（必須）・区分（店舗/会社）。**データ範囲（スコープ）は権限編集から撤去**＝実際の対象店舗は**ユーザー割当**で各ユーザーに紐付け（店舗ロール＝店舗ピッカー／会社ロール＝全社）。③に**「🔗 ユーザー割当で店舗を紐付け・割当 →」ボタン**を用意し、店舗ロール作成後そのまま割当画面へ遷移できる（新規作成中は未保存破棄を確認）。 さらに③に**「このロールの割当ユーザー×対象店舗」プレビュー**（ユーザー割当で設定した実店舗＋未設定ステータス：設定済み✅／未設定⚠を表示）を出し、権限設定側で紐付けの状況・漏れを把握できる（閲覧のみ・編集は割当で）。

- **編集モードのステップ番号は ①権限内容を編集／②基本設定**（①ベース選択は隠れるため）。新規作成は ①ベース／②内容／③基本。
- 区分はベースの既定。ベースと変えると**整合警告**を表示（データ範囲は割当で決まるため権限編集では扱わない）。
- **エディタは1画面に統合**：一覧のセルクリックも行の「権限編集」も**同じ権限編集（ウィザード）**に着地。
- **権限編集の戻り導線は来た元＋「権限一覧へ」の2ボタン**：役割サマリ経由→「← 役割サマリへ」＋「← 権限一覧へ」／割当編集画面経由→「← 割当編集へ」＋「← 権限一覧へ」／権限一覧経由→「← 権限一覧へ」のみ。保存/キャンセルとも来た元へ戻る。セルは**該当カテゴリへズーム（展開）**。粒度は カテゴリ→機能→指標 をこの1画面で完結（旧アコーディオン詳細画面は統合により撤去）。

### ロールに画面（機能）権限を追加する
- **既存画面を付与**：一覧でセルクリック→ウィザードの該当カテゴリにズーム、または行の「権限編集」で②グリッドから `なし→閲覧/操作` に変更。
- **画面内の内訳（機能）・指標**：権限編集②で画面名をクリックしてインライン展開し、機能のセグメント（なし/閲覧/操作）／指標の〔表示〕〔非表示〕で設定。
- **分析画面の指標／タグ**：行の「指標一覧（指標管理）」で母集合を選定。
- ※ システムに無い**新しい画面カテゴリ自体の追加**はプロダクト側のマスタ機能（本モックの権限設定UIの対象外）。新画面は各ロールに既定「なし」で並び、上記で付与する。

## 画面構成

権限一覧の上部ナビは用途で2セットに分離:
- **権限の設定** … `権限編集` ／ `指標一覧`

**指標一覧・指標条件設定**は**店舗/会社タブ**で切替（🏬店舗指標/🏢会社指標＝全社集計）。**全ロールで両タブを表示**（会社ロールも店舗ロールも両方見える）。会社指標は6系統25指標（売上/仕入・原価/人件費/固定費/利益の全社集計＋店舗横断ランキング・構成比）。

会社指標の**機密バッジ＝店舗ロールへの開示推奨度（2段階）**として使う（バッジは「テンプレ適用時のおすすめ開示レベル」）：
- **🔸店長まで（ベンチマーク系）**：店長が自店を全社平均と比べるのに有用な全社指標（全社の客数・客単価・売上前年比・来店組数・全店平均客単価・廃棄ロス・総労働時間・人時売上高）。**店長・エリア長は既定ON／従業員（現場）は既定OFF**。
- **🔒経営機密（経営の内側）**：全社の売上金額・仕入額・原価率・人件費・地代家賃・販促費・固定費合計・粗利・各利益・FL/FLR率・店舗別ランキング・業態別構成比。**全店舗ロールで既定OFF**。会社全体の営業利益などはここに含まれ初期は見えない。
- 会社ロール（経理/人事）は会社指標を既定ON。**いずれも管理者が後から変更可**（付与＝全社集計値の閲覧であって他店明細の行スコープ解放ではない）。

指標条件設定②編集の「＋条件を追加」は**上下ボタンとも共通のモーダル入力**（末尾追加＋該当行スクロール&ハイライト）。既存ルールの編集/削除は下段のインライン行で行う（追加＝モーダル・編集＝インラインで一貫）。
- **割当・俯瞰** … `役割サマリ` ／ `ユーザー割当`

**役割サマリ**は各ロールの権限量に加え、**「割当ユーザー N名（⚠未設定 検知）」列＋「割当へ →」ショートカット**を持つ（ロール起点で誰が持つか把握→割当へ遷移。付与の実体は割当画面に集約＝二重管理を避ける）。
- **「割当へ」はそのロールの割当ユーザーだけに絞り込み表示**（全件でどれが該当か不明を解消。バナー＋「すべて表示」で解除・該当チップ強調）。**管理者一覧を直接開いた時は全件のまま**。
- **戻り導線**：役割サマリ経由の時は**「← 役割サマリへ」＋「← 権限一覧へ」の2ボタン**を出す（サマリにも一覧にも戻れる）。直接アクセス時は「← 権限一覧へ」のみ。

## 権限一覧マトリクス

ロール × 権限カテゴリの一覧。**一覧画面権限 / 設定画面権限** の2タブで出し分け（各タブはその群のカテゴリのみ表示）。

- **一覧画面権限**（青）＝ 一覧・分析画面の閲覧可否
- **設定画面権限**（緑）＝ 設定画面の操作可否

左端は **区分列**（会社／店舗）を行にまたがり表示。権限名・操作列は固定幅でロックし、カテゴリ列が増えると横スクロール。

**オーナー（全権）は一覧から除外**：全権固定で編集対象外のため、マトリクスは設定対象の会社/店舗ロールのみ。オーナー専用機能（**指標管理／付与先を指定**）はヘッダーの「👑 オーナー操作」ドロップダウン（右端）に集約。

### セル表示
- **一覧画面**：`閲覧`（見られる）／`一部`（一部のみ）／`—`（なし）
- **設定画面**：`操作`＋`閲覧可`（操作は閲覧を内包）／`閲覧可`（見るだけ）／`—`（なし）
- **「一部」** ＝ カテゴリ内の複数機能で設定が混在している状態。
- **セルをクリック** → 統合ウィザードを開き、そのロール×カテゴリへ**ズーム展開**（オーナーは固定のため対象外）。
- 折りたたみ時の内訳セルは列名に沿った許可数（一覧＝`閲覧 N/総数`、設定＝`操作 N/総数`）。

### 表示ロール選択（タブの上）
- タブ（一覧/設定）の**上**に「👤 表示ロール」ハードプルダウン。**両タブ共通で効く行スコープ**。
- 大別**2グループ（会社ロール／店舗ロール）**＋内訳ロールを複数選択（オーナーは一覧対象外のため非表示）。
- グループ見出しで一括トグル、全て／クリア。階層は **行（ロール）→ 領域（タブ）→ 列（列表示）**。

### 列の表示（ツールバー）
- **列プルダウン**（表外固定・横スクロールで動かない）：`全て`＋各カテゴリを複数選択。クリア／閉じる、未選択は内訳1列。
- 既定で**主要画面の列を表示**（重要度順）。列順は選択順に従う。
- 列選択は `localStorage` に**自動記憶** → 次回アクセス時も同じ列（名前付き保存ビューは廃止）。

## 権限編集（統合ウィザード）

- 権限編集は**統合ウィザード1画面**（旧アコーディオン詳細画面は撤去）。②権限内容で **一覧画面権限 / 設定画面権限** を扱い、画面名クリックで同一画面に深掘り。
- **詳細はボタン様式で統一**（チェックボックス廃止）：機能＝`操作/閲覧/なし` の3状態セグメント（操作=登録・編集・削除を含む・操作詳細トグルは撤去）、**閲覧可能指標＝各項目〔表示〕〔非表示〕**、本部の指標一覧は**読取chip**。**閲覧可能指標はトグルで開閉**（既定は折りたたみ・件数のみ、開いた時に出る）→ 閲覧画面は既定で「閲覧／なし」のみのクリーン表示。
- **操作＝登録・編集・削除を一体（3状態：なし/閲覧/操作）**：登録できる＝編集もできる＝削除もできる（書き込み全般）。旧「操作詳細（登録/編集/削除トグル）」と「全て（削除）」は撤去。破壊的な削除は実行時の確認ダイアログで担保。
- **並び順の統一**：店舗ロールは権限順(エリア長→店長→従業員)。一覧マトリクス・権限編集・列ツールバーは重要度順(DEFAULT_COLS→残りCATEGORIES順)で共通。カテゴリ内の機能は序列順＝操作:大元設定→中設定(連動)→一覧→分析(予実)／閲覧:分析→一覧→設定確認。
- **表示＝実態の整合**：一覧の状態と権限編集の点灯が全カテゴリで一致（screenSegを群対応化。view群の満点＝閲覧。全118件 不一致0）。
- **単一機能カテゴリは重複を解消**：カテゴリ段と機能段で同じ `閲覧/なし` を二重表示しない（カテゴリ段に一本化し、トグル展開＝内訳・指標のみ）。
- **エディタは1画面に統合・別画面化しない**：セルは該当カテゴリへズーム、ボタンは先頭から、着地は同一画面（旧アコーディオン view-edit はコードごと撤去）。
- **ダッシュボード＝ランディング画面**：閲覧は全ロール常時可（閲覧可否設定なし＝「閲覧固定（ランディング）」）。一方カスタマイズは操作なので設定画面権限に **「ダッシュボード（カスタマイズ）」列**を持つ。**ダッシュボードの閲覧可能指標＝そのロールの指標一覧を自動反映（読取）**：管理者が指標管理／UIカスタマイズで閲覧可能にした指標が、ロールのダッシュボード権限に反映される。
- 新規実装は ★★ バッジ。

### 機能の集約・整理（過剰な機能別権限を解消）

「設定編集の可否だけで十分な画面を機能別に細分化しすぎない」方針で集約（総機能 **65→53**）:
- **従業員管理 7→2**：登録・編集・勤怠取込・時間記録・人件費・履歴・打刻を「従業員の管理」1つ（設定編集の可否）に統合。**給与のみ機密のため分離**。
- **予算設定 7→2**：連動サブ設定（単月費用/売上構成比/曜日別バイアス）を「月次予算設定（ウィザード）」に内包、月次＋年間を「予算一覧」に統合。**実在しない「予実管理カレンダー」は削除**（予実は集計の「予実対比（日次）」「年間予実対比」の2つが正）。
- **ダッシュボード（カスタマイズ）4→2**：ウィジェット＋DnD編集（店舗/会社）を「UIカスタマイズ」1つに統合、カスタムタグ管理は分離。

### 閲覧専用画面の扱い

- **閲覧画面には「操作詳細」を出さない**：閲覧の唯一の操作＝エクスポートは**廃止**し、ダウンロード/抽出は**「ダウンロード」カテゴリ（設定画面権限）で一括制御**（画面別エクスポートは持たない）。閲覧画面は「閲覧／なし」のみ。
- **閲覧トグルは"細かく設定できる時だけ"出す**：分析・一覧画面のうち**画面別／指標別の閲覧可否を細かく設定すべきもの**（売上分析・仕入分析・詳細分析・集計 等）は展開トグルを残し、展開すると**画面別の閲覧/なし＋指標の表示/非表示**を設定できる。**1層の閲覧/なしスイッチだけで担える画面**（従業員閲覧・締め状況閲覧 等）は**トグルを出さない**。
- **空トグルを解消**：展開しても中身（操作詳細・指標・複数機能）が無いカテゴリは**矢印を出さず・クリック展開しない**（「機能なし」表示を廃止）。

### 権限レベル（機能の性質ごと）

機能の種別で取りうる段階が異なる:

**大前提＝「閲覧できるか／操作できるか（閲覧を含む）」の3状態**（`なし ⊂ 閲覧 ⊂ 操作`）。**操作＝登録・編集・削除を含む書き込み全般**（削除だけ分けない）。旧「全て（削除）」と「操作詳細（登録/編集/削除トグル）」は**撤去**（3状態と重複するため）。破壊的な削除は**実際に削除する時の確認ダイアログ**で担保。

| 種別 | 段階 | 例 |
|---|---|---|
| 常時(base) | 閲覧固定 | （常時表示の閲覧） |
| 閲覧専用(view) | なし / 閲覧 | 分析・レポート・各種閲覧 |
| 設定(edit) | なし / 閲覧 / **操作**（登録・編集・削除を含む） | 納品書・棚卸・従業員管理 など |
| 管理(admin) | なし / 閲覧 / 設定可 | 会社情報・お支払い・連携設定・ユーザー など |
| 実行(action) | なし / 実行可 | ダウンロード 等 |

- **選択UIの点灯**：`操作`（または`設定可`）を選ぶと下位の`閲覧`も同時点灯（操作⊃閲覧の可視化）。
- **重要な内訳は個別設定**：カテゴリ内に別々に権限管理すべき機能（例：従業員管理の「給与」、小口現金の「経費項目設定」）がある場合はトグルで開き、内訳ごとに なし/閲覧/操作 を設定。単純な画面（内訳1つ・一括で足りる）はトグルなしで3状態スイッチのみ。
- **実行(action)** は「実行できる／できない」のため閲覧のみは設けない（2択）。

## ロール構成

- **オーナー** … 全権固定（編集不可）。全権の付与先を指名可。
- **会社ロール（全社・本部＝職務分掌）** … 経理担当・人事担当など。設定権限を職務に応じて付与。
- **店舗ロール（店舗現場）** … 店長・従業員・エリア長。
  - **店長**：自店舗の運用を操作（登録・編集・削除を含む＝3状態の「操作」）。**初期設定のロール**で、割当で**複数店舗の紐付けも可能**。
  - **エリア長**：**現時点で店長と権限差はない**（既定は店長と同じ）。**あくまで顧客都合でカスタマイズするロール名**であり、"上位"というわけではない。差別化が要る場合は**権限編集で画面を絞る/広げる＋掌握店舗（スコープ）**で行う。
  - **従業員**：閲覧中心（操作は最小）。

スコープ規則（軸は独立・割当で上書き可・機密は指標でゲート）：
- **区分（画面アクセス：店舗/会社）とデータ範囲（自店舗/指定店舗群/全社）は独立した別軸**＝組み合わせ自由（店舗ロール＋全社、会社ロール＋自店舗 等も可）。ロール定義の区分・範囲は**既定値（テンプレ）**にすぎない。
- **実際のスコープは割当時に束縛**（`user_roles.scope_type`＋`store_ids`）。同一人物でも割当で「複数店舗所属／全社」を持てる（例：従業員が複数店舗、従業員に会社ロール、データ範囲＝会社）。
- 区分/範囲をベース既定から変えるのは**意図的な変更として許可**（ブロックせず助言警告＋監査）。
- **2種類を区別する**：①**行データのスコープ遮断**（店長=自店舗の他店データを構造的に見せない）は**ハード強制で据え置き**。②**指標カタログの可視性**（会社営業利益などを指標一覧に出すか）は**既定OFF＋管理者オプトイン**＝店舗ロールにも会社指標を付与できる（全社集計/機密company指標は店舗ロールでは既定OFF、管理者が意図的にONにできる。ハードブロックはしない）。指標を付与しても①により他店の生データ行は依然見えない。

## ユーザー割当（管理者一覧）

### 適用の方向性（確定）
権限はユーザーに直接ではなく、**ロール経由**で適用する：`権限 → ロール → ユーザー`。**割当は「既存権限（ロール）を人に適用する」場所**であって権限を作り込む場ではない（RBAC＝定義は少数・共有／適用は多数。人ごとの権限増殖を防ぐ）。

- **適用対象＝経営管理にログインする管理者**（従業員台帳の `master` 連動アカウント）。**全従業員ではなく「ログインさせる人」**が対象。従業員台帳は所属店舗・役職を引く参照元＋ログイン者の選定元。
- **管理者一覧から選択してロールを割当**（複数可＝権限を合算＝和集合＝兼務対応。例：店長＋経理）。
- **割当画面の操作の意味**：ロールの**チェック＝既存ロールの付与**（新規作成ではない）／各ロールの**「権限編集 →」＝そのロール自体の権限を修正**（持つ全員に反映）／**「＋権限を新規作成」＝一覧に無い新ロールを作る**（この画面から作る時は①ベースが「プレーンテンプレートから作成」既定で開く）。
- **店舗ロール（店長/エリア長）は割当時に対象店舗を束縛**＝有効範囲を限定。会社ロールは全社。
- **権限設定との相互遷移**：ユーザー割当（割当編集画面）の各ロール行に「**権限編集 →**」（既登録ロールの権限編集画面へ）＋下部に「**＋ 権限を新規作成 →**」（新規登録の権限設定ページへ）を配置。権限編集③にも「ユーザー割当で店舗を紐付け・割当 →」＝双方向に行き来できる。
- **店舗紐付けはユーザー割当に一本化**：店舗ロールを付与したユーザーに、**専用の遷移画面**（モーダルでなく `view-assignedit`）で**対象店舗を紐付け**（店長=自店舗1つ／エリア長=複数選択／**先頭の「全店」チェックで一括選択**も可）。実効スコープの真実＝`user_roles.store_ids`はここ。有効範囲列にその人の実店舗を表示（例「店舗（渋谷店）」「指定店舗群（3店舗：…）」「全店（N店舗）」）。権限編集②基本設定は**権限名＋区分のみ**（データ範囲＝スコープの選択自体を撤去）。スコープは割当が唯一の真実（二重管理を完全排除）。
- **有効範囲** 列＝スコープ（自店舗／全社／指定店舗群）を表示。未割当は「なし」。行の「指標一覧（N）」ボタンに各ロールの閲覧可能指標数を表示し、指標一覧設定が一覧で把握可。
- **権限編集③の割当プレビュー**：店舗ロールの権限編集で、そのロールが**掌握する店舗の集約（店舗数＋店舗名）**を表示（人の一覧は出さない）。店舗未設定の保持者がいる時のみ⚠警告。右下に「🔗 ユーザー割当で紐付け →」（プレビュー無し時は区分の下に左詰）。
- **権限編集⇄割当の遷移（保存確認）**：「ユーザー割当で紐付け →」は正規の確認モーダル（キャンセル／保存せず移動／**保存して移動**）。**遷移先は賢く**：既存ロールは**そのロールを持つユーザーの割当編集画面**へ、保持者のいない新規ロールは**役割サマリ（新規行をハイライト）**へ。「保存して移動」は**権限名が未入力なら先へ進めない**（アラート）。
- 割当編集画面 → 権限設定へも遷移可（各ロール行「権限編集 →」／下部「＋権限を新規作成」。割当画面からの新規作成は①ベースが「プレーンテンプレートから作成」既定）。

### 本番実装の宿題
`user_roles`（ユーザー↔ロール多対多）＋スコープ束縛／適用タイミング（即時 or publish・ログイン中反映）／監査ログ（付与・剥奪）／ロックアウト防止（最後のオーナーを外せない）／既存 `master`・`manager`・`shop_manager` から新ロールへの移行マッピング。

## 公開・キャッシュ対策

GitHub Pages（**Actions ビルド**）で push 毎に自動デプロイ。常に最新が見えるよう恒久対策を実装:
- **Pages のビルド方式は「GitHub Actions」必須**：以前は legacy（ブランチ直配信）で **main の生ファイルが配信され、Action の sed が無視**＝`__BUILD__` が `__BUILD_PLACEHOLDER__` のまま未置換 → 自動更新が常に無効＝**全員が古いキャッシュを見続ける不具合**だった（2026-06-29 に `build_type=workflow` へ切替て解消）。**legacy に戻さないこと**。
- **ビルド版数の自動注入**：デプロイ時に Actions がコミットSHAを `__BUILD__` へ埋め込む（`pages.yml` の sed ステップ）。ライブで `window.__BUILD__='<sha>'` が実SHAになっていることを確認（`__BUILD_PLACEHOLDER__` のままなら配信方式が誤り）。
- **ネットワーク優先の自動更新**：起動時に最新HTMLを `no-store` 取得し、版数差があればその場で差し替え＋再初期化（1回ガードでループ防止）。
- **Service Worker解除＋キャッシュ破棄**、no-cache メタ。
- **右下に `build xxxxxxx`** を可視表示（最新確認用）。
- 切替直後の旧キャッシュ版は自動更新が効かないため、利用者は**一度だけハードリロード**で実SHA版を取得 → 以後は自動更新。
