# 検証プラン・合格基準（権限設定 本番実装）

各フェーズの「何を・どう検証し・どこで合格とするか」。合格点は**定量で判定可能**にする。全フェーズ共通で**フィーチャーフラグでON/OFF・即ロールバック**を前提。

## 全フェーズ共通（横断）
| 検証項目 | 方法 | 合格点 |
|---|---|---|
| ロールバック可能性 | フラグOFF→挙動確認 | フラグOFFで即・完全に従来挙動へ戻る |
| 回帰（既存機能） | 既存テスト/スモーク | 既存テスト全green・本番本体に破壊的変更0 |
| 受入の事前定義 | 各フェーズ着手前 | 合格点を着手前に確定（後出ししない） |

## P0：基盤（保存＋シャドー）
| 検証項目 | 方法 | 合格点 |
|---|---|---|
| 保存の往復 | UI→API→DB→再表示の単体/結合テスト | 保存内容が100%一致で再現（level/ops/指標allowlist） |
| 辞書バリデーション | 不明feature_key・tier不整合を投入 | 不正キー/不整合は400で弾く（取りこぼし0） |
| 実効権限の正しさ | 複数ロール和集合のテストケース | max(level)・OR(ops) が全ケースで期待一致 |
| キャッシュ invalidate | 権限変更→再取得 | 古い値が残らない／更新が次取得で反映 |
| シャドー差分 | 1〜2週シャドーlog集計（feature_key×ロール別） | 過剰ブロック（new=拒否, actual=許可）が主要画面で0・全体で合意閾値以下 |
| 実挙動の不変 | 本番挙動の回帰確認 | 判定未接続を確認（ユーザー影響0） |

## P1：閲覧判定
| 検証項目 | 方法 | 合格点 |
|---|---|---|
| 閲覧可否の正誤 | ロール別に対象画面の表示/非表示を実データ確認 | 見えるべき=100%見える／見えないべき=0%見える |
| 誤ブロック | シャドー→fail-close段階切替で観測 | 誤ブロック=0（業務停止0・緊急ロールバック0） |
| 段階切替 | fail-open(log)→fail-close | シャドー差分0収束を確認してから実拒否へ |

## P2／P2.5：操作・削除判定
| 検証項目 | 方法 | 合格点 |
|---|---|---|
| 操作可否 | 付与/未付与ロールで create/update 試行 | 付与のみ成功・未付与=403が100% |
| 案C連動 | 登録/編集の連動を操作テスト | 登録許可→編集も許可・操作なし→削除も無し が仕様一致 |
| 削除(P2.5) | 削除を付与/未付与・確認有無で試行 | 削除は付与ロール＆確認経由のみ成功・他は403／監査記録100% |

## P3：スコープ強制（最重要）
| 検証項目 | 方法 | 合格点 |
|---|---|---|
| 越境（情報漏洩） | 他店ID直指定・横断クエリで他店データ取得を試行 | 他店データ取得=0件／API他店指定=403 or 空（全モデル） |
| 件数比較 | for_user 結果 vs 現状を比較ログ | 想定外の消失/過剰露出=0（現場確認込み） |
| 業務影響 | 現場ロールで必要データの可視性確認 | 必要データ消失=0 |
| 性能 | 負荷・クエリ計測（INDEX/N+1） | レスポンス劣化が許容内（例 p95 +10%以内）・N+1=0 |

## P4：指標カタログ＋越境防御
| 検証項目 | 方法 | 合格点 |
|---|---|---|
| allowlist | ロール別の指標応答を期待allowlistとdiff | allowlist外の指標=応答0件 |
| 機密越境 | 店舗ロールで全社集計/機密company指標を全経路試行 | 店舗ロール応答に全社集計/機密company指標=0件（L1〜L8） |
| 指標メタ前提 | metrics整備の充足確認 | sensitivity/scope未設定の指標=fail-closed（既定company）で漏れ0 |

## P5：運用機構
| 検証項目 | 方法 | 合格点 |
|---|---|---|
| 監査網羅 | 各変更操作→audit_logs確認 | grant/revoke/delegate/scope変更が100%記録 |
| ロックアウト防止 | 最後のオーナー/全権の削除・降格を試行 | 不可（ブロック）＝100% |
| publish反映 | 下書き→適用→次リクエスト | 適用後の反映が次リクエストで100%・ログイン中も反映 |
| 移行一致 | 既存master/manager/shop_manager→新ロールの判定diff | 移行前後で判定結果一致=100%（差分0） |

## 判定の運用
- 各エピックの受入条件に**この表の合格点を貼り**、満たして初めて次フェーズへ。
- シャドー/比較ログは**日次集計**して0収束をトラッキング（憶測でなく実測）。
- 合否は「OK/NG＋数値（例：越境0件/2週間）」で記録。
