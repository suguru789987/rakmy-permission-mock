# 変更履歴（CHANGELOG）

ラクミー経営管理 権限設定モックの主な変更。新しい順。

## 2026-06-29（キャッシュ機構の根本修正）
- **「反映されない」が繰り返された真因を特定・解消**：GitHub Pages が legacy（ブランチ直配信）で main の生ファイルを配信し、ビルド版数を注入する Action の `sed` が無視＝`__BUILD__` が `__BUILD_PLACEHOLDER__` のまま未置換 → 自動更新（キャッシュ破棄）が常に無効＝全員が古いキャッシュを見続けていた。`build_type=workflow`（GitHub Actions ビルド）へ切替＋再デプロイし、ライブの `window.__BUILD__` が実SHAに置換されることを確認。以後はデプロイ毎に自動でキャッシュ破棄・最新反映。
- 付随で判明：直前の「操作詳細ボタンが点灯しない」は data-on 修正済みだが古いキャッシュ版を見ていたため未点灯に見えていた（コードは正しい）。利用者は一度だけハードリロードで実SHA版を取得。

## 2026-06-29（連動の向き是正＋ボタン点灯修正）
- **操作詳細の連動を「登録(親)→編集・削除」に確定**：登録を許可すると編集・削除も連動して許可、登録をなしにすると全てなし。編集・削除は個別調整可。当初「削除⊃編集⊃登録」で実装したが、ユーザー明示指示「登録ができれば編集や削除も」を正として是正。
- **操作詳細「許可/なし」・指標「表示/非表示」ボタンが点灯しないバグ修正**：ハイライトCSSがdata-v限定で、これらdata-v無しボタンに色が付かず機能していないように見えた→data-on属性＋CSS追加（許可/表示=緑、なし/非表示=グレー）。状態自体は変化していた。

## 2026-06-29（連動・トグル・アコーディオン撤去）
- **screenSeg群非対応バグ修正**：一覧＝閲覧可なのに権限編集でview群カテゴリの「閲覧」が未点灯（'op'→2がview段の閲覧1に当たらず）を群対応で解消。全ロール×全カテゴリ**118件 不一致0**（一覧と権限編集の点灯が一致）。
- **操作詳細のCRUD連動（削除⊃編集⊃登録）**：上位を許可すると下位も許可／下位をなしにすると上位もなし。「登録だけ許可・編集なし」等の不整合状態を防止。
- **操作詳細・閲覧可能指標をトグル折りたたみ**：既定は折りたたみ（バー＋件数のみ）、開いた時に操作項目／指標が出る。閲覧画面は既定で「閲覧／なし」のみのクリーン表示。
- **旧アコーディオン編集（view-edit）を完全撤去**：HTML＋関連関数（renderBody/renderCatList/renderCatDetail/featRowHTML/setCatVal/setAllCats/openCat/closeCat/setFeat/toggleOp/toggleExp/setEditTab/toggleSub/toggleItem/setItemsAll/filterItems/applyFilters/catSel/featRange/editRole/saveRole 等・約190行）を削除し統合ウィザードに一本化。セル遷移(gotoCatEdit)は維持。
- **設計方針の確定**：権限編集は別画面に出し分けず統合維持＋フォーカス（セルは該当カテゴリへズーム）。閲覧画面は操作詳細トグルで整理（エクスポート等は開いた時のみ）。

## 2026-06-29（権限編集の統合・ボタン様式・表示整合）
- **エディタ統合**：セル／「権限編集」ボタンとも唯一の統合ウィザードへ着地。セルは該当カテゴリへ**ズーム展開**。カテゴリ→機能→操作詳細→指標を1画面で完結（旧アコーディオン詳細は撤去）。編集モードのステップ番号は①権限内容／②基本設定。
- **詳細をボタン様式に統一**（チェックボックス廃止）：操作詳細＝操作項目を縦並び＋各行〔許可〕〔なし〕、指標（閲覧可能指標）＝各項目〔表示〕〔非表示〕、読取カタログ＝chip。**閲覧可能指標はトグルで開閉**（既定折りたたみ・件数のみ）。**単一機能カテゴリの「閲覧/なし」二重表示を解消**。
- **catStatus根本修正**：操作(2)を「全て(3)でない」ため閲覧と誤表示／lock(閲覧固定)をなしと誤判定する件を解消。全ロール×全カテゴリ**118件 不一致0**（一覧表示＝機能実態が一致。店長等の操作機能が「操作+閲覧可」と正表示）。
- **ダッシュボード＝ランディング画面**：閲覧は全ロール常時可（閲覧可否設定なし＝「閲覧固定（ランディング）」）。カスタマイズは操作なので設定画面権限に「ダッシュボード（カスタマイズ）」列。**ダッシュボードの閲覧可能指標＝ロールの指標一覧を自動反映（読取）**＝管理者が指標管理/UIカスタマイズで設定したものがダッシュボード権限に反映。
- **指標列設定を一覧に反映**：各ロールの「指標一覧（N）」ボタンに閲覧可能な指標数を表示。
- 呼称統一：指標カタログ→**指標一覧**／カタログ管理→**指標管理**。オーナー専用操作を**👑 オーナー操作**ドロップダウン（右端）に集約。系統見出しから「トランザクション：」を撤去。
